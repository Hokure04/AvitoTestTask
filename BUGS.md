## BUG-1. Некорректный формат успешного ответа при создании объявления

**Описание:**  
При успешном создании объявления API возвращает только поле `status`, а не объект созданного объявления

**Шаги воспроизведения:**  
1. Отправить POST-запрос на `POST /api/1/item` с валидным телом
2. Получить ответ сервера

**Ожидаемый результат:**  
Сервер должен возвращать объект созданного объявления с полями `id`, `sellerId`, `name`, `price`, `statistics`, `createdAt`

**Фактический результат:**  
Сервер возвращает `200 OK` и сообщение об успешном сохранении объявления


## BUG-2. API позволяет создать объявление с отрицательным значением price

**Описание:**  
Сервис принимает отрицательное значение `price` и успешно создаёт объявление

**Шаги воспроизведения:**  
1. Отправить POST-запрос на `POST /api/1/item`
2. Передать валидное тело объявления с отрицательным значением `price`

**Ожидаемый результат:**  
Сервер должен вернуть `400 Bad Request`. Объявление не должно создаваться

**Фактический результат:**  
Сервер возвращает `200 OK` и объявление создаётся


## BUG-3. API позволяет создать объявление со значением price выше диапазона integer

**Описание:**  
Сервис принимает значение `price`, превышающее диапазон `Integer`, и успешно создаёт объявление

**Шаги воспроизведения:**  
1. Отправить POST-запрос на `POST /api/1/item`
2. Передать валидное тело объявления со значением `price` больше `2147483647`

**Ожидаемый результат:**  
Сервер должен вернуть `400 Bad Request`. Объявление не должно создаваться

**Фактический результат:**  
Сервер возвращает `200 OK` и объявление создаётся


## BUG-4. API принимает отрицательный sellerId при получении списка объявлений

**Описание:**  
При передаче отрицательного `sellerId` сервис возвращает успешный ответ

**Шаги воспроизведения:**  
1. Отправить GET-запрос на `GET /api/1/{sellerId}/item`
2. Передать отрицательный `sellerId`, например `-1`

**Ожидаемый результат:**  
Сервер должен вернуть `400 Bad Request`

**Фактический результат:**  
Сервер возвращает `200 OK` и список объявлений

**Описание:**

**Фактический результат:**

## BUG-6. Статистика удалённого объявления остаётся доступной

**Описание:**  
После удаления объявления его стаистика продолжает быть доступной по идентификатору

**Шаги воспроизведения:**  
1. Создать объявление
2. Удалить объявление по `id`
3. Запросить статистику удалённого объявления

**Ожидаемый результат:**  
Сервер должен вернуть `404 Not Found`

**Фактический результат:**  
Сервер возвращает `200 OK` и статистику удалённого объявления
